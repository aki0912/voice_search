# TDDガイド

## 原則
- 失敗するテストから始める
- 実装は最小で通す
- リファクタは意味があるときにだけ

## 既存の赤テスト
- `TranscriptSearchServiceTests`
  - 単語検索
  - フレーズ検索
  - 辞書補正
  - 部分一致
  - 時間→最近接単語

## 追加すべきテスト（優先順）
1. 空入力・空クエリ
2. 大文字小文字混在
3. 多言語表記ゆれ（長音・濁点・全角半角）
4. 語の重複ヒット時の順序安定性
5. 大量データ時の上限件数

- `swift test` が通る
- コアAPIの互換性を崩さない
